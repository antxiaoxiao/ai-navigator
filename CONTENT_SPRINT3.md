## 一、职位选修内容方向

### 方向 1：UX 日常工作提效（以 Figma 真实工作流为主轴）

按 UX 设计师的完整工作流顺序排列，覆盖每一个可以被 AI 优化的环节。

---

#### 阶段一：项目启动与需求理解

- [ ] **需求对齐前的思路整理**
  拿到模糊需求时，用 AI 帮你快速提炼核心问题、识别信息缺口，生成需要向 PM 确认的问题清单。避免开工后返工。
  工具：Claude
  模版：`"我收到了以下设计需求：[粘贴需求]。请帮我：1. 提炼核心设计问题 2. 列出信息缺口 3. 生成需要向 PM 确认的5个关键问题"`

- [ ] **Precondition Check 清单生成**
  设计开始前，用 AI 生成当前任务的前置条件检查清单，确保设计依赖的内容（数据、边界、用户场景）都已明确。
  工具：Claude
  模版：`"我正在设计[功能名称]，目标用户是[用户群]，使用场景是[场景]。请帮我生成一份设计开始前的 Precondition Checklist"`

- [ ] **竞品分析快速框架生成**
  将竞品截图或功能描述喂给 AI，快速生成结构化的竞品分析框架，提炼差异点和设计机会。
  工具：Claude（支持图片上传）
  模版：`"请分析以下竞品的[功能/界面]，从用户体验角度指出：1. 设计优点 2. 潜在问题 3. 我们的差异化机会"`

- [ ] **用户旅程地图初稿生成**
  描述用户场景和痛点，让 AI 生成用户旅程地图的文字框架，再在 Figma 里可视化。
  工具：Claude + FigJam
  模版：`"目标用户是[画像]，核心任务是[任务]，请生成一份用户旅程地图，包含：阶段/行为/想法/痛点/机会点"`

---

#### 阶段二：设计执行

- [ ] **Design System 组件规范快速生成**
  将现有 Design System 的规则描述给 AI，让其生成新组件的规范文档初稿，包含命名规则、使用场景、禁止用法。
  工具：Claude
  模版：`"我们的 Design System 有以下规则：[粘贴规范]。请根据这个风格，为[新组件名称]生成规范文档"`

- [ ] **基于 Design System 生成新组件**
  将组件规范和现有组件截图喂给 AI，生成新组件的结构描述和 Figma 搭建步骤，或直接生成对应的 React 代码。
  工具：Claude（描述结构）/ Claude Code（生成代码）/ Figma AI
  模版：`"根据我们的 Design System 规范[粘贴规范]，请描述如何搭建一个[组件名]，需要包含哪些 variants 和 properties"`

- [ ] **关键页面衍生状态批量生成**
  主流程页面完成后，用 AI 生成所有需要设计的衍生状态清单，避免遗漏。
  工具：Claude
  模版：`"我完成了[页面名称]的主流程设计，请列出所有需要额外设计的状态：empty state / error state / loading state / disabled state / success state 等，并描述每个状态的触发条件"`

- [ ] **Figma 自动命名和图层整理**
  用 Figma AI 或插件对混乱的图层结构进行自动命名和整理，提高文件可读性和交接效率。
  工具：Figma AI / Rename It 插件
  步骤：选中所有图层 → 使用 Rename It 按规则批量重命名 → 用 Figma AI 检查命名一致性

- [ ] **响应式布局规则生成**
  描述设计意图，让 AI 生成移动端、平板、桌面端的适配规则，指导 Figma Auto Layout 设置。
  工具：Claude
  模版：`"这是我的桌面端设计[描述]，请生成响应式适配规则，说明在768px和375px断点下各元素如何变化"`

#### 阶段三：文字与内容

- [ ] **UI 文案撰写与优化**
  将界面中的按钮文字、提示语、错误信息等交给 AI 撰写或优化，确保语气一致、表达清晰。
  工具：Claude
  模版：`"请为以下 UI 场景撰写文案，语气要[友好/专业/简洁]，品牌调性参考：[品牌描述]。场景：[空状态/报错/成功提示/引导文案]"`

- [ ] **基于品牌手册生成文案**
  将品牌手册上传给 AI，让其生成符合品牌声音和调性的界面文案。
  工具：Claude（文件上传）
  步骤：上传品牌手册 PDF → 描述需要的文案场景 → AI 生成符合品牌规范的多个版本

- [ ] **多语言文案快速生成**
  将 UI 文案交给 AI 翻译，并检查是否符合目标语言的表达习惯（不只是直译）。
  工具：Claude
  模版：`"请将以下 UI 文案翻译成[语言]，注意本地化表达习惯，避免直译。原文：[文案]"`

- [ ] **Microcopy 系统化检查**
  将所有界面文案导出给 AI，检查整体一致性——语气、称谓、标点、术语是否统一。
  工具：Claude
  模版：`"以下是我们产品所有的 UI 文案，请检查：1. 语气是否一致 2. 同一概念是否用了不同词汇 3. 标点符号是否统一：[粘贴所有文案]"`

---

#### 阶段四：测试与验证

- [ ] **A/B 测试方案生成**
  描述设计假设，让 AI 生成 A/B 测试的完整方案——包括测试变量、成功指标、样本量估算。
  工具：Claude
  模版：`"我想测试[设计假设]，目标是提升[指标]。请生成 A/B 测试方案，包含：测试变量/对照组/实验组/成功指标/最小样本量建议"`

- [ ] **用户测试脚本生成**
  根据设计目标和测试重点，让 AI 生成用户测试访谈脚本，包含开场白、任务描述、追问问题。
  工具：Claude
  模版：`"我需要测试[功能/流程]，目标用户是[画像]，主要想验证[假设]。请生成用户测试脚本，包含：开场白/热身问题/5个核心任务/追问问题"`

- [ ] **可用性问题分析与优先级排序**
  将用户测试的观察记录喂给 AI，生成问题分类、严重程度评估和修改优先级建议。
  工具：Claude
  模版：`"以下是用户测试中观察到的问题记录：[粘贴记录]。请按严重程度分类，并给出修改优先级建议（高/中/低）"`

#### 阶段五：交付与协作

- [ ] **Handoff 文档自动生成**
  设计完成后，描述设计意图和交互逻辑，让 AI 生成开发交付说明，减少手动撰写时间。
  工具：Claude
  模版：`"以下是[页面/功能]的设计说明，请生成开发 Handoff 文档，包含：功能描述/交互逻辑/边界情况/动画规格/需要开发注意的细节"`

- [ ] **设计评审 PPT 内容生成**
  将设计方案的核心内容描述给 AI，生成设计评审演示文档的内容框架和文案。
  工具：Claude
  模版：`"我需要向[受众]做设计评审，方案是[描述]。请生成演示文档的内容大纲，包含：问题背景/设计目标/方案说明/取舍决策/下一步计划"`

- [ ] **设计决策记录（Design Decision Log）**
  将每次重要的设计决策和背后的思考喂给 AI，生成结构化的设计决策记录，方便日后追溯。
  工具：Claude
  模版：`"我做了以下设计决策：[描述决策]，原因是：[描述原因]，放弃的方案是：[描述]。请整理成标准的设计决策记录格式"`

### 方向 2：影响团队——AI 辅助下的新 Sprint 工作流

基于 UX 想改变团队工作方式的需求，给出在 AI 辅助下的新工作流和具体实操。

- [ ] **Sprint Planning 用 AI 做工作量估算**
  将待设计的功能列表喂给 AI，生成设计工作量的初步估算和风险提示。

- [ ] **用研访谈记录的批量整理与洞察提炼**
  将多份用研记录喂给 AI，自动合并、分类、提炼 key findings 和设计机会点。
  工具：Claude（支持长文档）

- [ ] **设计 QA Checklist 自动生成**
  根据功能描述，让 AI 生成设计 QA 检查清单，确保交付前覆盖所有检查项。

- [ ] **团队 AI 工作流 Demo 方案**
  如何用 15 分钟向团队展示 AI 的真实价值——用真实数据（时间对比）而不是概念说服团队。
  工具：Claude + 真实工作样本 + 计时器

- [ ] **建立团队共用的 Prompt 模版库**
  收集团队高频使用的 prompt，建立共用模版库，减少每个人重复摸索的时间。

### 方向 3：转型 PM——AI 辅助下的 PM 新工作流

- [ ] **需求文档（PRD）初稿生成**
  给 AI 描述功能背景、用户需求和业务目标，生成 PRD 框架和初稿。
  模版：`"请为以下功能生成 PRD 初稿：背景[描述]/用户问题[描述]/业务目标[描述]，包含：功能概述/用户故事/验收标准/边界情况"`

- [ ] **用 AI 做功能优先级判断（RICE 框架）**
  将功能列表喂给 AI，按 Reach / Impact / Confidence / Effort 做初步评分和排序。

- [ ] **用 AI 写 Stakeholder 沟通文档**
  将技术方案或设计决策翻译成非技术受众能理解的语言，用于向管理层或客户汇报。

- [ ] **数据报告洞察提炼**
  将产品数据报告喂给 AI，快速提炼关键指标变化的原因和下一步建议。

- [ ] **前端基础——用 Claude Code 生成可运行原型**
  把设计好的界面用 Claude Code 生成 React 代码，快速验证交互逻辑，不需要等开发资源。

### 方向 4：创业者——AI 辅助下的 0 到 1

- [ ] **商业想法快速验证**
  用 AI 扮演"魔鬼代言人"，指出商业假设中的风险和盲点，避免过早投入。
  模版：`"我的商业想法是[描述]，目标用户是[描述]，盈利模式是[描述]。请从投资人角度，指出最大的3个风险和你会问的关键问题"`

- [ ] **竞品地图生成**
  描述产品方向，让 AI 生成竞品格局地图——直接竞品、间接竞品、替代方案。

- [ ] **用户访谈问题生成**
  在没有用户研究背景的情况下，用 AI 生成高质量的用户访谈问题，避免引导性提问。

- [ ] **Landing Page 文案生成**
  用 AI 生成产品首页的核心文案——价值主张、功能卖点、CTA 文案。

- [ ] **用 Claude Code 做 MVP**
  没有开发背景，如何用 Claude Code 把产品想法变成可运行的原型，完成第一次真实验证。

## 二、内容制作标准（六要素）

每条实操内容必须包含：

```
1. 场景描述    — 什么情况下用（具体工作场景，不抽象）
2. 使用工具    — 具体软件名称（Figma / Claude / Cursor 等）
3. 操作步骤    — 分步骤，每步一句话，不超过 5 步
4. 模版或起点  — 可直接复制使用的 prompt / 文件 / 链接
5. 预计时间    — 15 分钟 / 1 小时 / 本周内
6. 成功标志    — 做完之后你应该得到什么
```

## 三、内容来源策略

```
第一层：策展外部资源
        找真实被验证的模版和案例，注明来源，加编辑点评
        "为什么这个模版好用，在什么场景下用"

第二层：Claude 个性化调整
        用户看到通用模版，可点击"根据我的情况调整"
        Claude 根据用户职位和行为信号生成贴合版本

第三层：用户自己沉淀
        用过的好模版收藏进"我的模版库"
```
